<h1 align="center"> Автосервис</h1>
<p align="center">
  <b>Комплексная нетиповая 1C-конфигурация для автоматизации автосервиса</b><br>
</p>



Автоматизированная система учёта для сети автосервисов на платформе «1С:Предприятие». Проект охватывает оперативный, бухгалтерский и кадровый учёт, расчёт заработной платы, аналитическую отчётность и сервисные функции.

---

##  Цели проекта

- Автоматизировать ключевые процессы автосервиса: приём заказов, складской учёт, взаиморасчёты, контроль исполнения работ.
- Обеспечить корректный расчёт заработной платы с учётом графиков, больничных и премий.
- Реализовать полноценный бухгалтерский учёт (план счетов, проводки, отчётность).
- Предоставить руководству аналитику по продажам, остаткам, финансовым результатам и начислениям.
- Упростить ввод данных за счёт загрузки прайс‑листов из Excel.
- Создать понятный интерфейс с разделением по подсистемам и группами команд.

---

##  Функциональность

### Оперативный учёт (MVP, реализовано)

**Справочники:**
- Номенклатура (типы: товар, материал, услуга, материал клиента) с контролем групп и автоматическим типом.
- Контрагенты, договоры, организации (с проверками ИНН по типу юрлица/ИП).
- Сотрудники, должности, склады, кассы, статьи затрат, автомобили, цвета, модели и др.

**Документы:**
- «Поступление товаров и услуг» — приёмка и учёт услуг поставщиков.
- «Заказ‑наряд» — регистрация заявок, подбор услуг и материалов клиента.
- «Реализация» — продажа товаров/услуг, списание материалов.
- «Приходный кассовый ордер» и «Расходный кассовый ордер» — наличные расчёты.
- «Учётная политика» — выбор способа списания запасов (FEFO / по средней).

**Регистры накопления:**
- «Товары на складах» — остатки и движение ТМЦ.
- «Заказы клиентов» — контроль исполнения заказов.
- «Продажи» — обороты по номенклатуре, контрагентам, сотрудникам.
- «Учёт затрат» — затраты по статьям.
- «Денежные средства» — остатки по кассам.
- «Материалы заказчиков» — движение материалов, предоставленных клиентом.

**Отчёты (на СКД):**
- «Товары на складах» (остатки, приход, расход, итоги, выделение отрицательных значений красным).
- «Продажи» (по мастерам, клиентам, услугам, сортировка по сумме).
- «Финансовый результат» (доходы, расходы, прибыль/убыток).
- «Движение наличных денег».
- «Материалы заказчика».

### Бухгалтерский учёт и расчёт зарплаты (часть 2, требования описаны, частично реализовано)

- План счетов и регистр бухгалтерии с автоматическими проводками.
- Расчёт зарплаты: оклад, больничный, премии; регистры расчёта и виды расчёта.
- Отчёты: «Оборотно‑сальдовая ведомость по счёту», «По начислению заработной платы».

### Сервисные функции

- Загрузка цен номенклатуры из Excel (обработка с валидацией и сообщениями об ошибках).
- Мобильное бронирование (функционал запланирован, требует отдельного ТЗ).
- Настройка интерфейса: подсистемы, группы команд, начальная страница, стили, иконки, логотип, заставка.
- Функциональная опция «Использовать склады» для динамического управления видимостью объектов.

##  Технические решения

- **Регистры накопления** (остатки/обороты) — для количественно‑суммовых показателей.
- **Регистры сведений** — для данных, изменяющихся во времени (цены, графики, учётная политика).
- **Регистры расчёта** — для корректного расчёта зарплаты с учётом вытесняющих видов расчёта.
- **СКД (схема компоновки данных)** — для всех отчётов (отборы, группировки, итоги, условное оформление).
- **Автозаполнение и контроль ввода:** ответственный = текущий пользователь; проверки ИНН, запрет отрицательных остатков, контроль типов и групп.
- **Бухгалтерские проводки** — формируются автоматически при проведении документов; субконто заполняется по данным документа.
